### 命令式编程
**特点**：使用`print`、`+`、`if`等语句实时更改程序状态，按顺序执行代码，Python 是典型解释型语言
- 优点：易于使用和调试，代码简洁易懂，可通过打印中间变量、使用 Python 调试工具排查问题。
- 缺点：效率较低，Python 解释器会产生额外开销，尤其在多 GPU 场景下易成瓶颈；需保存中间变量，占用内存。
### 符号式编程
- **特点**：先定义计算流程，再编译成可执行程序，最后输入数据执行，支持大量优化。
- **步骤**：
    1. 定义计算流程，如通过函数生成`add`和`fancy_func`的代码字符串。
    2. 编译流程，使用`compile`函数将代码字符串编译为可执行程序。
    3. 执行程序，调用`exec`执行编译后的程序。
- **优缺点**：
    - 优点：运行效率高，可跳过 Python 解释器，消除多 GPU 场景下的性能瓶颈；编译器能优化代码（如将`(a+b)+(c+d)`直接优化为具体结果），还可释放无用内存；易于移植，可转换为非 Python 格式运行。
    - 缺点：灵活性低，需先完整定义流程再编译，调试难度较大。
```python
def add_():
    return '''
def add(a, b):
    return a + b
'''

def fancy_func_():
    return '''
def fancy_func(a, b, c, d):
    e = add(a, b)
    f = add(c, d)
    g = add(e, f)
    return g
'''

def evoke_():
    return add_() + fancy_func_() + 'print(fancy_func(1, 2, 3, 4))'

prog = evoke_()
print(prog)
y = compile(prog, '', 'exec')
exec(y)
```

|对比维度|命令式编程|符号式编程|
|---|---|---|
|执行方式|实时顺序执行|先定义流程、编译，再执行|
|易用性|高，代码直观|低，需提前规划流程|
|调试难度|低，可查看中间变量|高，编译后难追踪问题|
|性能|低，解释器有开销|高，编译器可优化|
|移植性|低，依赖 Python 环境|高，可转换为非 Python 格式|
## 混合式编程
### 核心思想

结合命令式编程的易用性与符号式编程的高效性，开发时用命令式编程调试，部署时转换为符号式编程提升性能。
